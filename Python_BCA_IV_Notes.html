UNIT 1 — Python Introduction & Core Language

1.1 Introduction to Python

  • Python is a high-level, interpreted, general-purpose programming language.
  • Created by Guido van Rossum; first released in 1991.
  • Syntax is close to the English language — easy to read & write.
  • Supports multiple paradigms: Object-Oriented, Functional, Procedural.
  • Used in: Web Dev, Data Science, AI/ML, Automation, Scripting, etc.
Real-Life: Think of Python as the "Swiss Army knife" of programming — you can build websites, automate boring tasks, analyze data, and even control robots with it.

1.2 Features, Advantages & Disadvantages

Key Features

#FeatureQuick Note
1Simple & EasyClean syntax, close to English; no semicolons/brackets needed
2InterpretedExecuted line-by-line; no compilation step; errors shown immediately
3High-LevelNo manual memory management; garbage collection handled automatically

5Platform Independent"Write Once, Run Anywhere" (Windows, Linux, macOS)

7Large Library Support1,40,000+ packages (NumPy, Pandas, Matplotlib, etc.)

Advantages

  • Easy Syntax → Fewer lines of code compared to C/Java.
  • Fast Development → Simple syntax + ready-made libraries = rapid prototyping.
  • Large Community → Tutorials, docs, Stack Overflow support.
  • Multi-paradigm → Procedural + OOP + Functional flexibility.

Disadvantages

  • Slower Execution → Interpreted + dynamic typing = slower than C/C++/Java. Not ideal for real-time systems or game engines.
  • Not Ideal for Mobile Dev → Limited mobile frameworks; Java/Kotlin/Swift preferred for mobile apps.
  • Higher Memory Usage → Dynamic typing + garbage collection consumes more RAM. Not great for embedded/memory-constrained devices.

1.4 Python Implementation (Variants)

How Python Works Internally (Implementation Process):

  1. Writing Source Code → Python programs written in .py files (human-readable).
  2. Compilation to Bytecode → Interpreter compiles source code → bytecode (intermediate, low-level representation; stored in .pyc files inside __pycache__).
  3. Python Virtual Machine (PVM) → Executes bytecode line by line → makes Python platform independent (same bytecode runs on different systems).
Real-Life: It's like translating a book (source code) into a universal shorthand (bytecode), and then a reader (PVM) on any device can understand and read it aloud.

Types of Python Implementations

Base of Differentiation: Programming Language written in & Running Environment (VM)

JythonJavaJVMCompiles to Java bytecode; integrates with Java libraries
IronPythonC#.NET CLRIntegrates with .NET framework libraries
PyPyRPythonPyPy VMUses JIT compiler → often faster than CPython
Exam Tip: CPython (default, C), Jython (Java, JVM), IronPython (.NET, C#), PyPy (faster, JIT). Differentiation = language + VM.

  • Steps: Download installer → Run installer → Follow on-screen instructions.

1.6 The Python Interpreter

  • A software program that executes Python code.
  • Reads code line by line → converts to bytecode → runs on PVM.
  • If an error occurs → stops immediately and displays the error message.

Working Steps

  1. Reads Source Code: Reads the .py file.
  2. Converts to Bytecode: Source code → low-level, platform-independent bytecode.
  3. Execution by PVM: Bytecode executed line by line.
  4. Error Handling: Stops & displays error immediately on encountering one.

Two Modes of Interpreter

AspectInteractive ModeScript Mode
ExecutionCommands one at a timeEntire program at once
ResultsShown immediatelyAfter full execution

Types of Python Interpreters

  • CPython – Default and most widely used interpreter.
  • Jython – Runs on Java Virtual Machine.
  • IronPython – Runs on .NET framework.
  • PyPy – Faster interpreter using JIT compilation.

Advantages of Python Interpreter

  • Easy debugging
  • Platform independent execution
  • Immediate error detection
  • Supports interactive programming

The lexical structure defines the basic building blocks of Python. It explains how a Python program is written and how the interpreter breaks the code into tokens.

Understanding lexical structure is essential for writing correct and error-free Python programs.

1.9 Variables

  • A variable is a name given to a memory location that stores a value.
  • The value can change during program execution, hence the name variable.
  • Created automatically on assignment — no explicit declaration needed (dynamic typing).

Variable Naming Rules

  1. Must start with a letter (a-z, A-Z) or underscore _.
  2. Cannot start with a number.
  3. Can contain only letters, digits, and underscore.
  4. Cannot use Python keywords.
  5. Case-sensitive: Age ≠ age.

Valid: total = 50, _totalMarks = 90, studentName = "Aman"

Invalid: 2value (starts with digit), class (keyword)

Variable Assignment Techniques

# Basic assignment
x = 10
name = "Parth"

# Multiple assignment (different values)
a, b, c = 10, 20, 30

# Same value to multiple variables
x = y = z = 5

# Reassignment (value can change anytime)
x = 10
x = 20   # x is now 20

# Deleting a variable
del x    # x no longer exists; using x after this causes NameError

1.15 Conditional (Decision-Making) Statements

Control flow statements control the order in which statements are executed. They decide:

  • Which statement will run
  • When it will run
  • How many times it will run

Conditional statements are used to execute different code based on conditions.

Real-Life: If it rains → take umbrella. Else → go normally. That's conditional logic!

Types of Conditional Statements

  1. if statement
  2. if-else statement
  3. if-elif-else ladder
  4. Nested if statement
  5. Short-hand if / Ternary operator

a) if Statement

Executes a block of code only when the condition is True.

age = 20
if age >= 18:
    print("You are eligible to vote")
# Output: You are eligible to vote

b) if-else Statement

if block runs when condition is True; else block runs when condition is False.

num = 5
if num % 2 == 0:
    print("Even number")
else:
    print("Odd number")   # This runs

c) if-elif-else Ladder

Used when multiple conditions need to be checked sequentially. Only the first True block executes.

marks = 85

if marks >= 90:
    print("Grade A")
elif marks >= 75:
    print("Grade B")   # This runs (first true condition)
elif marks >= 50:
    print("Grade C")
else:
    print("Fail")

d) Nested if Statement

An if statement inside another if statement.

age = 20
citizen = True

if age >= 18:
    if citizen:
        print("You can vote")

e) Short-Hand if & Ternary Operator

# Short-hand if (one-liner)
a = 10
b = 5
if a > b: print("a is greater")

# Ternary Operator (short-hand if-else)
num = 10
result = "Even" if num % 2 == 0 else "Odd"
print(result)   # Even

1.16 Looping (Iteration) Statements

Looping = repeating a set of statements again and again until a condition is satisfied.

Why loops?

  • Repeat tasks automatically
  • Work with large data efficiently
  • Reduce code repetition
  • Improve program efficiency

Python supports two types of loops: for and while.

Real-Life: Taking attendance of 60 students — you repeat the same question 60 times. That's a loop!

a) for Loop (known iterations)

Used when the number of repetitions is known.

# Print 0 to 4
for i in range(5):
    print(i)      # 0, 1, 2, 3, 4

# Iterating over a list
fruits = ["apple", "banana", "mango"]
for fruit in fruits:
    print(fruit)

range() Function

Syntax: range(start, stop, step)

CallGenerates
range(5)0, 1, 2, 3, 4
range(1, 6)1, 2, 3, 4, 5
range(1, 10, 2)1, 3, 5, 7, 9
range(10, 0, -1)10, 9, 8, ..., 1

b) while Loop (unknown iterations)

Used when the number of repetitions is NOT known. Runs as long as the condition is True.

i = 1
while i <= 5:
    print(i)   # 1, 2, 3, 4, 5
    i += 1

Infinite Loop

A loop that never stops. Use with caution!

# WARNING: This runs forever
# while True:
#     print("Hello")

c) Nested Loops

A loop inside another loop.

for i in range(3):
    for j in range(2):
        print(i, j)
# Output: 0 0, 0 1, 1 0, 1 1, 2 0, 2 1

d) Loop with else

The else block runs after the loop finishes normally (NOT interrupted by break).

for i in range(5):
    print(i)
else:
    print("Loop completed!")  # Runs after loop ends normally

1.17 Control Flow Statements (break / continue / pass)

These are jump/transfer statements used to control loop execution.

a) break — Stop the loop immediately

for i in range(1, 6):
    if i == 3:
        break       # Exit loop when i is 3
    print(i)
# Output: 1, 2

b) continue — Skip current iteration

for i in range(1, 6):
    if i == 3:
        continue    # Skip when i is 3
    print(i)
# Output: 1, 2, 4, 5

c) pass — Do nothing (placeholder)

for i in range(5):
    pass   # Placeholder - no error, does nothing

# Useful when you plan to write code later
def my_function():
    pass   # TODO: implement later

Quick Comparison Table

Exam Tip: break = stop loop. continue = skip iteration. pass = do nothing (placeholder). These three are always asked together in exams. Know code examples for each!

Q14. Explain == vs is with an example.

▶ Show Answer
== checks value equality (are the values same?).
is checks identity (are they the same object in memory?).
a = [1, 2, 3]
b = [1, 2, 3]
print(a == b)  # True  (same values)
print(a is b)  # False (different objects in memory)

Q19. Differentiate between break, continue, and pass.

▶ Show Answer
