3.1 Classes & Objects

  • Class = Blueprint. Object = Instance (has State, Behavior, Identity).
  • __init__() = constructor, runs automatically on object creation.
  • self = reference to current instance. Must be first param in all methods.
  • OOP advantages: code reusability, DRY, easier debugging, clear structure.
class Person:
    def __init__(self, name, age=18):  # Constructor + default value
        self.name = name
        self.age = age
    def greet(self):
        print(f"Hello, I am {self.name}")

p1 = Person("Emil", 25)
p1.greet()         # Hello, I am Emil

3.2 Inheritance & Polymorphism

Inheritance TypeDescription
SingleChild ← one Parent
MultipleChild ← Parent1 + Parent2
MultilevelGrandparent → Parent → Child
HierarchicalMultiple children ← one Parent
HybridCombination of above

super() calls parent methods without naming the parent class.

class Person:
    def __init__(self, fname):
        self.firstname = fname

class Student(Person):
    def __init__(self, fname, grade):
        super().__init__(fname)   # Call parent constructor
        self.grade = grade

Polymorphism

"Many forms" — same method/operator behaves differently.

Operator Overloading: 5+3=8, "Hi"+"!"="Hi!", [1]+[2]=[1,2]

Method Overriding (Runtime): Child redefines parent's method.

class Animal:
    def speak(self): print("Animal speaks")
class Dog(Animal):
    def speak(self): print("Dog barks")  # Overrides parent

Dog().speak()  # Dog barks

3.3 Data Hiding & Metaclasses

AccessPrefixWhere Accessible
PublicNoneAnywhere
Protected_singleClass + child classes (convention)
Private__doubleOnly within class

Name Mangling: __var becomes _ClassName__var internally.

Metaclass: Creates other classes. Chain: Metaclass → Class → Object. Default = type.

class Student:
    def __init__(self):
        self.__marks = 90      # Private
    def display(self):
        print(self.__marks)    # Access via method

s = Student()
# s.__marks                   # ERROR!
s.display()                    # 90
print(s._Student__marks)       # 90 (name mangling, not recommended)

3.4 Exception Handling

KeywordPurpose
tryCode that may cause error
exceptHandle the error
finallyAlways runs (cleanup)
raiseManually trigger exception

Exception Propagation: Unhandled exception passes up to calling function.

Custom Exception: Must inherit from Exception.

# try-except-finally
try:
    num = 10 / 0
except ZeroDivisionError:
    print("Cannot divide by zero")
finally:
    print("Always runs")

# Multiple except
try:
    num = int(input("Enter: "))
    result = 10 / num
except ValueError:
    print("Invalid input")
except ZeroDivisionError:
    print("Division by zero!")

# Generic except
except Exception as e:
    print("Error:", e)

# User Defined Exception
class AgeError(Exception):
    pass

try:
    if age < 18:
        raise AgeError("Must be 18+")
except AgeError as e:
    print(e)
Exam Tip: Exception vs Error: Exceptions can be handled; Errors (like SyntaxError) usually cannot.

3.7 File I/O Operations

ModeMeaning
rRead (default). Error if not exists.
wWrite. Overwrites. Creates if not exists.
aAppend. Adds at end. Creates if not exists.
xCreate new. Error if exists.
r+/w+/a+Read+Write combos. w+ truncates; a+ no truncate.
bBinary mode (e.g. "rb", "wb")

Reading: f.read()=all, f.read(n)=n chars, f.readline()=one line, f.readlines()=all as list.

# Read
with open("data.txt", "r") as f:  # auto-closes!
    data = f.read()

# Write (overwrites)
with open("demo.txt", "w") as f:
    f.write("New content")

# Append
with open("demo.txt", "a") as f:
    f.write("\nAppended")

# Keyboard input
name = input("Enter name: ")
Exam Tip: After read(), cursor is at end — readline() returns empty. Use with to auto-close. w creates file if not exists but r gives error.
